### 実装段階で発生した環境依存の罠（次回も再発しうる）

| 罠 | 回避策 |
|---|---|
| `pnpm/action-setup@v4` の `version: 10` と `package.json` の `packageManager: pnpm@10.32.1` 重複指定で `ERR_PNPM_BAD_PM_VERSION` | action 側の `version` を消して `packageManager` フィールドに従わせる |
| Windows + git bash で **`jq` が利用不可** | hook スクリプトは Node.js (`process.stdin.on('data', ...)`) で stdin JSON parse |
| Windows bash で `BASE_PATH=/physics-base-synth pnpm build` が `BASE_PATH=C:/Program Files/Git/...` に変換される (MSYS path conv) | ローカル試験は `MSYS_NO_PATHCONV=1` を前置 |
| Svelte 5 `svelte/prefer-svelte-reactivity` rule が effect-local な `new Set<string>()` も警告 (false positive) | `// eslint-disable-next-line svelte/prefer-svelte-reactivity` で局所抑止 |
| sv create テンプレートに残った `eslint-plugin-storybook` import が、storybook 関連を消した後にエラー | `web/eslint.config.js` から該当行を削除 |
| `AudioWorkletProcessor` 型が TypeScript の `lib.dom.d.ts` に存在しない | `synth-processor.ts` 冒頭に `declare class AudioWorkletProcessor` |
| Claude Code の hook watcher は **セッション開始時に存在しなかった `.claude/`** を監視しない | `/hooks` で 1 度リロード、または再起動 |
| GitHub Pages のローカル build とデプロイ build で WASM ハッシュが異なる | `_app/immutable/nodes/<n>.js` から実 hash を抽出して 200 確認 |

### 推奨スコープ

仕様書 `01-overview.md`「Phase 2 への申し送り」と「非ゴール」表からの優先順位案：

| 候補 | 推奨 | 理由 |
|---|---|---|
| ポリフォニー (`VoicePool<const N>`) | **含める** | Phase 1 の単一前提を根本から再設計するタイミングで一緒にやる方が安い |
| Extended KS: fractional delay | **含める** | A1 で 2.3% のピッチ誤差は MVP の妥協。Phase 2 で解消 |
| `ParamDescriptor` + `params.json` コード生成 | **含める** | `ParamId` / `PARAM_IDS` 二重管理を Phase 2 中に増やすと負債が雪だるま化 |
| Extended KS: loss filter / pick position | 余裕があれば | 音色のリアリティ。fractional delay の後に乗せやすい |
| ボディ共鳴 (Body Resonator) | **Phase 3 へ送る** | IR convolution / modal / static IR の選択が Phase 2 で本筋から外れる |
| MIDI CC マッピング (pitch bend / mod / sustain) | 余裕があれば | UI 改修込み。Phase 2 終盤での追加候補 |
| プリセット保存・ロード | **Phase 3 以降** | `localStorage` / IndexedDB / file 等の戦略決定が必要 |
| hold note stack | **含める（軽微）** | Phase 2 でポリフォニー化する流れに付随 |

### 着手前に答えを出すべき調査項目

1. **fractional delay 方式**: Lagrange interpolation (4-tap) vs Thiran allpass — CPU 負荷とピッチ誤差のトレードオフ
2. **Voice stealing アルゴリズム**: oldest / quietest / round-robin。`KarplusStrong::energy` で quietest が低コスト
3. **C ABI の API 変化方針**: `synth_note_on(handle, midi, vel)` を維持し内部で voice id 割当 vs voice id 引数追加
4. **`params.json` コード生成 tooling**: build.rs (Rust) + Vite plugin (TS) or 別途 codegen スクリプト
5. **WASM SIMD**: `target-feature=+simd128` 安定化状況、ポリフォニー 8 voice での効果
6. **ブラウザ互換性 (2026-05 時点)**: iOS Safari / Firefox の AudioWorklet・Web MIDI の最新状況、render quantum 128 固定の現状
