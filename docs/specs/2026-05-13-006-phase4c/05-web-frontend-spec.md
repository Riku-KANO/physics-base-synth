# 05. Web フロントエンド仕様（Phase 4c）

## 目的

Phase 4c で追加・変更する Svelte コンポーネント、TypeScript モジュール、AudioWorklet 拡張を定義する。Phase 1 / 2 / 3 / 4a / 4b で確立した既存コンポーネント（`StartButton` / `Keyboard` / `MidiSelect` / `ParamSlider` / `VoiceMeter` / `PolyphonyToggle` / `ModWheel` / `LfoSection` / `PresetSelector`）と既存ステート（`synth.svelte.ts` / `ui.svelte.ts` / `voice-state.svelte.ts` / `preset-store.svelte.ts`）はすべて維持し、本書では **Phase 4c で追加・変更する箇所のみ** 記述する。

Phase 4c は UI レベルでは「Piano プリセットの内部値が変わる（Step 19 の聴感チューニング結果）」のみで、新規コンポーネント追加なし / 新規 UI 露出なし（D81）。`__synthDev.measureProcessTime` API は Phase 4b で完成済のため変更なし、Phase 4c では Step 1 / Step 20 で API を呼び出すユーザー操作のみ行う。

### 1.2 Phase 4c の Piano エントリ（Step 19 聴感調整後の暫定値）

Step 19 の聴感チューニングで反復調整した結果として、以下のような値域で Piano プリセットを更新（概数、最終値は実装段階で確定）:

```typescript
{
  version: 1,
  name: 'Piano',
  createdAt: '2026-05-13T00:00:00.000Z',  // Phase 4c 着手日に更新
  instrument: 'piano',
  params: {
    damping: 0.998,         // Phase 4b と同等（Multi-string で sustain 強化されるため微調整余地あり）
    brightness: 0.50,       // Phase 4b 0.55 → 微減（Hertz hammer で高域既に増、補正）
    outputGain: 0.65,       // Phase 4b 0.7 → 微減（Multi-string で voice 内合算が増えるため）
    pickPosition: 0.13,     // Piano では未使用（hammer 経路）、互換用
    bodyWet: 0.50,          // Phase 4b 0.4 → 増（響板感強化）
  },
  lfo: { rate: 5.0, waveform: 'sine', pitchDepth: 0.0, brightnessDepth: 0.0, volumeDepth: 0.0 }
}
```

**注意**: 上記値は **暫定の聴感調整方向性**を示すもので、Step 19 のユーザー実機聴感で反復決定する（D82）。仕様書ドリブン開発の例外として「聴感調整値は実装段階で確定」を許容（Phase 4b D62 で予告した運用）。

## 2. `params.ts`（gen-params.mjs 出力）の Phase 4c 追加

```typescript
// === Phase 4c で生成される const (auto-generated by gen-params.mjs) ===

/// Piano の unison detuning 量 (cents、D72)
export const UNISON_DETUNE_CENTS_PIANO = 1.5;

/// Piano の sympathetic resonance amount (0.0..=1.0、D77)
export const SYMPATHETIC_AMOUNT_PIANO = 1.0;

/// Hammer cutoff Hz の上下限（D75）
export const HAMMER_CUTOFF_LOW_PIANO = 800;
export const HAMMER_CUTOFF_HIGH_PIANO = 5500;

/// 88 鍵 × f32 LUT (D78 / D79)
/// TypeScript 側は dsp-core 内部で参照されるのみで、フロントエンドからは直接使用しない
/// 露出する場合は `Readonly<number[]>` 型で 88 要素
export const INHARMONICITY_B_CURVE_PIANO: ReadonlyArray<number> = [
  // 88 値（Rust 側 params.rs と同期、A0 = index 0 から C8 = index 87 まで）
  3.1e-4, 2.9e-4, /* ... */ 5.6e-2,
];
```

TypeScript 側で Piano プリセットを表示する `factory-presets.ts` がこれらを参照することはなく、純粋に dsp-core 内部の WASM コード領域に埋め込まれる。`gen-params.mjs` の TypeScript 出力は型整合性の維持のみ目的。

## 8. Preset スキーマの互換性（localStorage v1 維持）

Phase 4c で Piano プリセットの **内部値**（`damping` / `brightness` / `outputGain` / `pickPosition` / `bodyWet`）が変更されるが、スキーマ自体は localStorage v1 のまま:

- 既存ユーザーが Phase 4b までに保存した User Preset は Phase 4c でも読み込み可能
- Phase 4b に保存された "My Piano Preset" 等の User Preset は **Phase 4c でも値が変わらず**動作する（Phase 4b の Piano エンジンと Phase 4c の Multi-string Piano エンジンで音色は異なるが、`params` 値の互換性は保たれる）
- Factory Preset の Piano エントリは Phase 4c リリース時に最新値で上書き（既存 User Preset は影響なし）

## 10. 実機聴感確認の手順（Step 18-19 で実施）

Step 18-19 で実機聴感確認を行う手順:

1. `pnpm dev` で Vite dev server 起動
2. ブラウザで `http://localhost:5173/` を開き、`Start` ボタン押下
3. PresetSelector で `Piano` を選択
4. キーボードで C4 / A4 / C5 / A5 / C6 等を試奏
5. **判定基準**（D82）:
   - Phase 4b との比較で「弦楽器寄り」感が改善している（Multi-string で beating + two-stage decay が聞き取れる）
   - 強打鍵時の打鍵 transient が Phase 4b より鮮明（Hertz hammer の効果）
   - Sustain ペダル ON で「ペダル音の余韻」が出る（Sympathetic resonance の効果）
   - 低音域（A1, C2）と高音域（C7, C8）でそれぞれ「ピアノらしさ」が出る（B(note) LUT の効果）
6. 不足があれば Piano プリセット値 (`damping` / `brightness` / `bodyWet` / 内部 `unison_detune_cents` / `sympathetic_amount` / `hammer_cutoff_*`) を反復調整 → Step 19 で再評価
7. 既存 7 楽器 (Default / Guitar / Bass / Sitar 等) も試奏し、regression なしを確認

ユーザー実機聴感達成までは Phase 4c 完了とみなさない（D82）。

## まとめ

Phase 4c の Web フロントエンド仕様は **UI レベルでの変更最小** が中核。新規コンポーネント / 新規ストア / 新規 action / 新規 UI 露出はすべてなし（D81）。`factory-presets.ts` の Piano エントリのみ Step 19 の聴感チューニング結果で値を更新、`gen-params.mjs` 出力に `INHARMONICITY_B_CURVE_PIANO` / `UNISON_DETUNE_CENTS_PIANO` / `SYMPATHETIC_AMOUNT_PIANO` を追加。`__synthDev.measureProcessTime` API は Phase 4b 完成済を使用、Step 1 / Step 20 でユーザー操作必須。Preset スキーマは localStorage v1 維持で既存 User Preset 互換性保たれる。`CLAUDE.md` 完了済みイテレーション欄を Phase 4c retrospective 完了時に追記。
